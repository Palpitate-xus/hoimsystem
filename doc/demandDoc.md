# 医院门诊信息管理系统需求文档

## 1. 系统目标

随着计算机技术的发展，计算机在医院系统中的应用逐渐普及，利用医院门诊信息管理系统进行医院的运营减少了人力、物力资源的浪费，提高了工作效率和医院的整体竞争力。

医院门诊信息管理系统是现代化医院运营的必要技术支撑和基础设施，实现医院信息管理系统的目的就是为了以更现代化、科学化、规范化的手段来加强医院中病人、医生、病房、科室和药品的管理，同时实现门诊挂号自动化、医生诊断科学化、病历系统化、取药自动化等，提高医院的工作效率，改进医疗质量。

---

## 2. 系统角色与权限

系统包含四种用户角色，各角色权限如下：

| 角色 | 权限说明 |
|:----:|:--------:|
| admin（管理员） | 拥有系统最高权限，可管理医生、病人、科室、药品、通知公告、收费记录等 |
| director（科室主任） | 拥有 doctor 权限，额外可管理本科室排班、发布通知等 |
| doctor（医生） | 可查看病人信息、开具处方、管理排班、查看病历等 |
| patient（病人） | 可预约挂号、查看挂号/预约记录、查看收费记录、查看通知等 |

---

## 3. 系统功能模块

系统按用户角色划分为以下功能模块：

### 3.1 登录模块（公共）

- 用户登录（支持密码加密传输）
- 用户注册（病人自助注册）
- 获取公钥（用于前端加密）
- 获取当前登录用户信息（权限、用户名、头像）
- 用户登出

### 3.2 管理员模块

- **医生管理**：医生信息的录入、维护与查询
- **病人管理**：病人信息的查询
- **科室管理**：科室信息的录入、维护与查询（科室名称、电话、位置、主任）
- **通知公告管理**：发布系统通知和公告，支持按角色定向推送
- **收费管理**：查看所有收费记录

### 3.3 病人模块

- **预约挂号**：查看医生排班，选择日期、科室、医生、时段进行预约
- **现场挂号**：查看当日医生排班并进行现场挂号
- **挂号/预约记录查询**：查看历史挂号和预约记录，支持取消
- **收费查询**：查看个人待缴费和已缴费记录，支持在线缴费

### 3.4 医生模块

- **医生注册**：管理员或主任录入医生信息并创建登录账号
- **排班管理**：设置每周出诊排班及号源数量
- **药品管理**：药品信息的录入、维护、查询及库存查询
- **处方管理**：为病人开具处方，系统自动扣减药品库存并生成收费记录
- **处方查询**：查看历史处方记录

## 4. 业务流程

![病人就医流程](../doc_assets/PatientAccessFlow.png)

<p align="center">图1 病人就医的主要流程</p>

### 4.1 就医流程

1. 病人注册/登录系统
2. 病人选择预约挂号或现场挂号
3. 系统判断医生号源，有剩余则挂号成功，否则失败
4. 病人按挂号时间就诊
5. 医生诊断后开具处方
6. 系统根据处方自动生成收费记录
7. 病人缴费后完成取药

### 4.2 挂号规则

- 现场挂号：选择当日出诊医生，判断是否有剩余号源
- 预约挂号：最多提前7天预约，不能预约当天的号
- 病人在同一天对同一医生的挂号/预约次数不能超过一次
- 预约可在就诊前取消

### 4.3 排班规则

- 科室主任或管理员为医生设置每周排班
- 排班包含：星期、时段、号源数量、是否专家号
- 病人根据排班信息进行挂号或预约

### 4.4 处方与收费规则

- 医生开具处方时选择病人和药品，填写数量
- 系统自动扣减药品库存
- 系统根据药品价格和数量自动计算收费金额
- 生成收费记录，状态为"未缴费"
- 病人缴费后，收费状态变为"已缴费"

## 5. 数据需求分析

### 5.1 实体及属性

| 实体 | 主要属性 | 说明 |
|:----:|:--------:|:----:|
| 用户 | 用户ID、用户名、密码、角色 | 系统登录账号 |
| 病人 | 病人ID、姓名、性别、身份证号、生日、手机号、地址、权限 | 病人基本信息 |
| 医生 | 医生ID、姓名、性别、科室ID、职称、学历、手机号、权限、用户ID | 医生基本信息 |
| 科室 | 科室ID、名称、电话、位置、主任医生ID | 科室信息 |
| 排班 | 排班ID、星期、时段、号源数、是否专家号、医生ID | 医生出诊排班 |
| 挂号 | 挂号UUID、序号、病人ID、医生ID、是否专家号、科室ID、时间、状态 | 现场挂号记录 |
| 预约 | 预约UUID、病人ID、医生ID、是否专家号、科室ID、偏好时段、预约日期、状态 | 预约挂号记录 |
| 处方 | 处方UUID、病人ID、医生ID | 医生开具的处方 |
| 处方药品 | 记录UUID、处方ID、药品ID、数量 | 处方与药品的关联 |
| 收费 | 收费UUID、创建时间、缴费时间、处方ID、金额、状态 | 收费记录 |
| 药品 | 药品ID、名称、库存、单价、过期日期、采购时间、供应商、备注 | 药品库存信息 |
| 通知 | 通知UUID、标题、内容、是否紧急、目标人群、发送时间、过期时间、阅读数、发布人ID | 系统通知公告 |
| 病历 | 病历UUID、就诊时间、医生ID、病人ID、症状、诊断结果 | 病人病历 |
| 违约记录 | 记录UUID、挂号ID | 病人违约记录 |

### 5.2 状态定义

**挂号状态：**
- 0 = 未就诊
- 1 = 已就诊
- 3 = 已取消

**预约状态：**
- 0 = 未就诊
- 1 = 已就诊
- 2 = 已取消

**收费状态：**
- 0 = 未缴费
- 1 = 已缴费

## 6. 业务规则及完整性约束

### 6.1 组织关系约束

- 一所医院由多个科室组成；一个科室有多名医生；一名医生只能属于一个科室。
- 一个病历只能对应一个病人，但是可以对应多个医生。
- 一个病人只能对应一个主治医生，但一个主治医生可以对应多个病人。

### 6.2 挂号约束

- 病人现场挂号时，如果医生号源已满则拒绝挂号。
- 病人在同一天的预约和挂号次数不能超过两次。
- 一个病人在同一时间段只能挂号一次。
- 预约挂号最多提前7天，不能预约当天的号。
- 在就诊前一天允许取消预约。

### 6.3 处方与收费约束

- 一张处方单对应一个病人编号，但可以包含多个药品。
- 一张处方单只能对应开一张发票，且只能划价一次。
- 缴费一次可以包含多张处方单，一张处方单缴费一次。
- 处方单的取药状态记录了该处方对应的病人是否取药完成。
- 在处方单的取药状态为未完成之前允许医生修改处方单，病人取药之后不允许修改。

### 6.4 药品库存约束

- 系统记录每种药品当前的库存数量，药品出库时系统自动减少库存数量。
- 系统自动记录药品的有效日期，在到期前1个月提醒药房工作人员补货，到期时提醒报废处理。

### 6.5 排班约束

- 一个医生在一个时间段只能接待一个病人（通过号源数控制）。
- 医生如果有事停诊需要至少提前一天通知管理员，由管理员发布停诊信息。

### 6.6 排队规则

- 排队序号由病人在挂号时系统自动生成。
- 对于预约病人，预约时间越早排队序号越小；对于现场挂号的病人，挂号时间越早排队序号越小。
- 在病人从系统获取编号时，系统自动提醒病人当前排队人数。
- 如果排队过号，依次向后顺延5个号。

### 6.7 违约处理

- 病人在一个月内三次预约挂号未取号，则暂停该病人预约资格一个月。
- 系统自动记录违约记录。

## 7. 非功能性需求

- **安全性**：用户密码需加密传输（RSA公钥加密），关键操作需身份验证
- **易用性**：系统界面应简洁直观，符合医院工作人员操作习惯
- **可扩展性**：系统架构应支持后续功能模块的扩展，如检查管理、住院管理等
- **可靠性**：核心数据（挂号、收费、处方）需保证数据一致性，支持事务回滚
