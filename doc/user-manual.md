## 一、系统概述

### 1.1 系统简介

HOIM(Hospital Outpatient Information Management)医院信息管理系统,覆盖 **门诊 / 住院 / 药房 / 收费 / 检验 / 体检 / 手术 / 随访** 9 大核心业务,支持 **11 种角色** 协同工作日结。

### 1.2 技术架构

| 层 | 组件 |
|:--|:--|
| 前端 | Vue 3 + Element Plus + Rspack(端口 8091) |
| 后端 | FastAPI + SQLAlchemy + SQLite/PostgreSQL(端口 8000) |
| 安全 | JWT + bcrypt + RBAC 11 角色 × 247 接口 |

### 1.3 界面布局

```
┌────────────┬─────────────────────────────────────────────┐
│            │  顶部工具栏: 用户头像 / 消息通知 / 退出     │
│   左侧     ├─────────────────────────────────────────────┤
│   导航     │                                             │
│   菜单     │  主内容区(表格/表单/图表)                   │
│   (角色    │                                             │
│    过滤)   │  左侧: 分页 / 表格操作按钮                  │
│            │                                             │
└────────────┴─────────────────────────────────────────────┘
```

**左侧菜单根据当前登录角色动态显示**,未授权模块不可见(也无法通过 URL 直接访问,后端会返回 403)。

## 二、登录与注册

### 2.1 登录系统

1. 浏览器打开 http://localhost:8091
2. 进入登录页,输入**账号**(身份证号 / 工号)和**密码**
3. 点击"登录"

| 角色 | 账号 | 密码 | 说明 |
|:--|:--|:--|:--|
| 管理员 | `admin` | `admin123` | 全部权限 |
| 科室主任 | `director01` | `123456` | 管理 + 本科室 |
| 医生 | `doc01` | `123456` | 诊疗工作 |
| 药师 | `pharmacist01` | `123456` | 药房工作 |
| 收费员 | `cashier01` | `123456` | 收费窗口 |
| 护士 | `nurse01` | `123456` | 护理工作 |
| 导诊员 | `guide01` | `123456` | 导诊台 |
| 检验技师 | `lab01` | `123456` | 检验科 |
| 患者 | `370101199001011234` | `123456` | 自助服务 |
| 超级管理员 | `super01` | `123456` | 全部权限(最高) |

### 2.2 患者注册

1. 登录页点击"注册账号"
2. 填写:**账号(身份证号同)**、密码、身份证号、姓名、性别、手机号、地址、出生日期
3. 提交后自动登录,角色为"患者"

### 2.3 退出登录

点击右上角用户头像 → "退出登录"(同时清 token 和前端缓存,不会残留 session)。

### 2.4 修改密码

当前版本通过**管理员**操作(`管理员模块 > 权限分配 > 重置密码`)。

## 三、管理员(admin)

> **核心职责**:系统全局管理 + 用户/科室/药品 + 系统级配置 + 数据运维

### 3.1 医生管理(`管理员 > 医生管理`)

- **新增医生**:点击"新增" → 填写工号/密码/姓名/职称/科室/权限(doctor 或 director)/学历 → 保存
- **编辑**:列表行"编辑"(可修改科室、职称、权限)
- **删除**:列表行"删除"(物理删除,谨慎!)

### 3.2 病人管理(`管理员 > 病人管理`)

- 表格显示全部注册患者,可按姓名/身份证搜索
- 点击"编辑"可修改:姓名/性别/手机号/地址/**过敏史**(重要!)
- **关键字段**:`过敏史`为逗号分隔,例如"青霉素,头孢,磺胺",处方开立时会触发过敏校验

### 3.3 科室管理(`管理员 > 科室管理`)

- 新增:填写科室名/电话/位置/主任
- 编辑:修改基本信息
- 删除:物理删除

### 3.4 通知公告(`管理员 > 通知公告`)

- 点击"新增通知" → 标题/内容/紧急标记/发送对象(全部/医生/患者)/过期时间
- 列表展示作者/发送时间/已读数
- 过期通知自动不展示

### 3.5 号源池管理(`管理员 > 号源池管理 → 号源调整`)

- 按科室看号源总数/医生数
- 点击"调整"可修改某排班的号源数

### 3.6 科研数据导出(`系统管理 > 科研数据导出`)

为科研项目提供 CSV 数据集下载:

| 可导出的表 | 说明 |
|:--|:--|
| patients | 患者基本信息(姓名/性别/出生日期/过敏史) |
| medical_records | 病历记录(主诉/诊断/时间) |
| prescriptions | 处方药品明细(药品名/数量/状态) |
| charges | 收费记录(金额/状态) |
| lab_results | 检验结果(项目/结果/异常标记) |
| surgeries | 手术记录(手术名/麻醉/状态) |
| exams | 体检结果(项目/数值/参考范围) |

- **脱敏模式(推荐开启)**:姓名/身份证/手机号自动 hash,不可逆但仍可关联
- **时间范围**:可指定 from/to,限定研究时间窗
- **全量数据包**:一键导出 7 张表的 ZIP(含 manifest 索引)
- **审计日志**:所有导出操作都会被记录(`/api/research/export/audit`),仅 admin 可查
- **权限**:admin ✅ director ✅,其他角色 403

### 3.7 报表统计(`统计报表 > 统计报表`)

- 门诊量(按日/周/月/科室/医生)
- 收费统计(收入/退费/药占比)
- 药品用量统计
- 医生工作量(病历/处方/检查)

### 3.8 操作日志(`系统管理 > 操作日志`)

- 按用户/动作/时间段搜索
- 全量审计,精确到秒

### 3.9 数据备份恢复(`系统管理 > 数据备份恢复`)

- 手动触发 SQLite 备份
- 历史备份列表,一键恢复

### 3.10 数据字典(`系统管理 > 数据字典`)

- 业务枚举维护(药品类别、检查类型等,供前端下拉)

### 3.11 系统参数(`系统管理 > 系统参数`)

- 配置项 key-value 管理(站点名、工作时长等)

### 3.12 权限分配(`系统管理 > 权限分配`)

- 用户列表(含密码重置/角色切换)

## 四、科室主任(director)

> **核心职责**:医生工作站 + 本科室事务(通知/排班调整)

### 4.1 医生排班(`医生工作站 > 医生排班`)

- 查看个人周排班 + 号源余量
- 与普通医生相同的操作界面

### 4.2 病历管理(`医生工作站 > 病历管理`)

- 写主诉/现病史/既往史/诊断/治疗
- 可使用**病历模板**(`电子病历 > 病历模板`)
- **权限**:只能修改自己创建的病历

### 4.3 处方管理(`医生工作站 > 处方管理`)

- 处方开立 → 药品选择 → 库存/过敏/抗菌分级自动校验
- 列表查看历史 + 取消(仅 status=0 可取消,已发药不可)

### 4.4 检查检验申请(`医生工作站 > 检查检验申请`)

- 选患者 → 选类型 → 勾项目 → 提交

### 4.5 考勤签到(`医生工作站 > 考勤签到`)

- 签到 → 系统记录 + 迟到判断(≥9:00 迟);签退

### 4.6 多学科会诊(`医生工作站 > 多学科会诊`)

- 发起 MDT:患者/参与科室/会诊意见
- 会诊结果模板

### 4.7 临床路径(`医生工作站 > 临床路径`)

- 入径/出径 + 阶段进度

### 4.8 通知公告(本科室发布)

- director 角色可在本科室内发布与优先级相同

### 4.9 报表查看

- 本科室工作量/收费/药品(与 admin 看到的维度相同)

## 五、医生(doctor)

> **核心职责**:诊间诊疗全流程

### 5.1 医生排班

- 查看个人周排班 + 号源余量

### 5.2 病历管理

- **新建**:选患者 → 填写主诉/现病史/查体/诊断/治疗 → 保存
- **编辑**:可编辑自己创建的病历(`medicalRecord/update` 具备 owner 校验)
- **模板**:使用病历模板快速填充

### 5.3 处方管理(带智能审核)

系统自动校验:

| 校验项 | 行为 |
|:--|:--|
| ⚠️ 库存不足 | `药品 X 库存不足` |
| ⚠️ 过敏史冲突 | `过敏史冲突:病人对 [X] 过敏` |
| ⚠️ 限制级抗菌药 | 需 director 审批 |
| ⚠️ 特殊使用级抗菌药 | 需 admin 审批 |

- **取消**:仅 status=0(待审核)可取消,取消后库存自动还原

### 5.4 检查检验申请

- 可给任意患者开立(临床需要)
- 紧急标记:是否为紧急(危急值)

### 5.5 考勤签到

- 上班签到 + 下班签退

### 5.6 多学科会诊(MDT)

- 发起会诊 / 录入结果

### 5.7 临床路径

- 患者入径 + 进度跟踪

## 六、药师(pharmacist)

> **核心职责**:药品/耗材管理 + 处方审核/发药/退药 + ADR 监测

### 6.1 药品管理(`药房管理 > 药品管理`)

- 新增药品:名称/库存/价格/效期/供应商/是否抗菌药级别
- 修改/列表查询(支持库存查询/低库存预警/近效期提醒)

| 预警类型 | 触发条件 |
|:--|:--|
| 低库存预警 | 库存 ≤ 10(可调阈值) |
| 近效期提醒 | 距效期 ≤ 30 天(可调阈值) |

### 6.2 处方审核与发药(`药房管理 > 处方审核与发药`)

闭环状态机:

```
待审核(0) → [药师审核] → 已审核(1) → [发药] → 已发药(2) → [退药] → 退药
```

- **审核**:列表 status=0 → 点击"审核" → 状态变 1
- **发药**:列表 status=1 → 点击"发药" → 状态变 2
- **退药**:退数量 ≤ 已发数量;退药后库存自动还原

### 6.3 库存盘点(`药房管理 > 库存盘点`)

- 实物盘点 → 输入实盘数 → 系统自动算盈亏 → 更新库存

### 6.4 处方点评(`药房管理 > 处方点评`)

- 点评已发药处方:合理/问题/建议

### 6.5 耗材管理(`药房管理 > 耗材管理`)

- 与药品管理 CRUD 模式相同

### 6.6 药品采购(`药房管理 > 药品采购`)

- 采购订单 CRUD + 供应商管理

### 6.7 ADR 不良反应监测(`药房管理 > ADR 监测`)

- 上报不良反应(关联药品)
- 修改状态(待审核/已确认/已处理)

### 6.8 低库存预警 / 近效期

- `药房管理 > 库存预警` 查看当前预警列表

## 七、收费员(cashier)

> **核心职责**:门诊收费窗口 + 窗口挂号/退号 + 日结对账

### 7.1 窗口挂号(`收费管理 > 窗口挂号`)

1. 刷卡/输入患者身份证
2. 选科室/医生/时段
3. 确认号源数量 > 0
4. 提交 → 系统扣减号源 + 生成挂号单

### 7.2 费用管理(`收费管理 > 费用管理`)

- 列表:已缴费/待缴费/已退费
- **缴费**:点击对应收费单 → 提交(状态 0→1)
- **退费**:已缴费单 → 退费(状态 1→2)

> ⚠️ 缴费后无法重复缴费,退费后无法重复退费。

### 7.3 发票管理(`收费管理 > 发票管理`)

- 开发票:关联已缴费收费单
- 打印发票(当前为 mock,无真实 PDF)

### 7.4 日结对账(`收费管理 > 日结对账`)

- 选择日期(默认当天)
- 汇总:已收/已退/待收 + 笔数

### 7.5 支付(online)

- 患者在线扫码支付(微信支付/支付宝/现金 mock)
- 支付成功后自动更新 charge 状态 0→1

### 7.6 住院结算(扩展)

- 住院费用结算(住院费用模块,需要 cashier 权限时)

## 八、护士(nurse)

> **核心职责**:临床护理 + 预检/分诊 + 入院出院办理

### 8.1 生命体征录入(`护士预检 > 生命体征录入`)

- 选患者/入院记录
- 录入:体温/血压(收缩/舒张)/脉搏/体重
- 实时同步到体温单

### 8.2 护理记录(护理工作站)

- 列表查询 + 新增/编辑(按 admission 过滤)
- 删除:列表行"删除"

### 8.3 体温记录

- 日期 + 时间点(如 08:00)
- 同一日期+时间点 `update if exist,else create`

### 8.4 预检分诊(分诊台协同)

- 体温/血压/心率快速录入
- 自动分级

### 8.5 入院登记(`住院管理 > 入院登记`)

1. 选患者(必须已注册)
2. 选病区 + 床位(校验床位是否已占用)
3. 输入主诉/入院诊断/押金
4. 提交 → **自动占床(床位 status 0→1) + 生成床位费费用**

> 校验:一个患者不可二次入院(需先出院)

### 8.6 出院结算(`住院管理 > 出院结算`)

1. 选在院患者(admission status=1)
2. 输入出院诊断
3. 提交 → **自动停止新开/已审核的医嘱 + 释放床位(status→0) + 生成出院小结**

### 8.7 护士工作站

- 执行医嘱(长期/临时):record execution
- 体温单管理

## 九、导诊员(guide)

> **核心职责**:患者引流 + 分诊台 + 候诊队列/巡视

### 9.1 智能导诊(患者自助也可)

- 输入症状关键词
- 系统匹配科室/医生(310+ 关键词库)

### 9.2 分诊台管理(`分诊台 → 创建`)

- 输入患者身份证 → 自动查到他/她
- 测量体温/血压 → 自动分级(1 危急 - 4 非急)
- 导诊到对应科室

### 9.3 候诊队列(`分诊台 → 候诊队列`)

- 按医生/科室看候诊列表
- 支持:叫号/跳过/过号/急诊插队

### 9.4 候诊巡视(`分诊台 → 候诊巡视`)

- 巡诊记录:巡诊时间/关注患者/处理措施

## 十、检验技师(lab_technician)

> **核心职责**:检验样本接收 + 结果录入 + 危急值审核

### 10.1 样本接收(`检验科 → 样本接收`)

- 输入申请单号 → 确认接收(状态: 待接收 → 已接收)
- 拒收:填写拒收原因

### 10.2 检查结果录入(`检验科 → 检查结果录入`)

- 选申请单 → 录入各项目结果
- **危急值判断**:
  - 血压:收缩 ≥180 或 ≤90 / 舒张 ≥110 或 ≤60
  - 血糖:≥16.7 或 ≤2.8
  - 心率:≥120 或 ≤50

危急值自动标记 `abnormal_flag = 1`。

### 10.3 结果审核(`检验科 → 审核`)

- 结果状态: 待审核 → 已审核

### 10.4 报告查看

- 查询患者历史检验结果
- 异常值高亮

## 十二、患者(patient)

> **核心职责**:全生命周期自助服务

### 12.1 智能导诊(`患者服务 > 智能导诊`)

- 输入症状 → 推荐科室+医生

### 12.2 预约挂号(`患者服务 > 预约挂号`)

1. 选科室/医生/日期(7 天内)/时段
2. 查看号源余量
3. 提交预约 → 系统扣减 1 个号源

> **违约规则**:30 天内违约 3 次 → 暂停预约 30 天
> 违约:预约日超时未报到(由报到时判定)

### 12.3 现场挂号(`患者服务 > 现场挂号`)

- 直接挂当天已放出号源的医生

### 12.4 报到签到(患者自助 kiosk)

- 输入身份证 + 预约 UUID → 生成排队号
- 支持违约判定

### 12.5 缴费管理(`患者服务 > 缴费管理`)

- 查看待缴费/已缴费/已退费
- 在线扫码支付(需收费员配合 mockNotify)

### 13.2 RBAC 实现方式

- 前端:路由 `meta.permissions` 过滤菜单(可见性)
- 后端:FastAPI `Depends(require_roles(...))`(强校验)
- 两道防线,绕过前端直调 API 仍会 403

### 13.3 状态机总览

**处方**: `0 待审核 → 1 已审核 → 2 已发药 | 3 已取消(返回库存)`
**号源**: `N → 挂号 -1 → 退号 +1`
**住院**: `0 待入院 → 1 在院 → 2 已出院 | 停药+释放床位`
**Charge**: `0 待缴费 → 1 已缴费 → 2 已退费`

## 十四、常见问题 FAQ

### Q1:登录提示"账户或密码不正确"?
- 确认账号是身份证号(患者)/工号(医生)
- 默认密码见 §2.1 表格
- 区分大小写

### Q2:菜单看不到某个功能?
- 角色权限限制,只显示已授权模块
- 直接 URL 访问也会返回 403

### Q3:库存显示负数?
- 当前库存检查仅在**处方开立**时校验,盘点可能有差异
- 使用 `药房管理 > 库存盘点` 校正

### Q4:预约时提示"号源已满"?
- 已有人占满,选其他时段
- 号源在**取消挂号 + 退号后自动返还**

### Q5:处方被"过敏史冲突"拒绝?
- 患者过敏史在 `管理员 > 病人管理` 修改后立即生效

### Q6:出院时报"病人不在院状态"?
- 患者已出院或未入院,需先办理入院

### Q7:收费员退费后,处方状态会变吗?
- 当前版本**不退联动**(设计约定:退费与临床分离)
- 如需作废处方,由医生手动取消

### Q8:如何新增导诊/检验/挂号员角色?
- 由管理员在 `权限分配` 中修改用户角色即可
- 角色关键字段:`guide` / `lab_technician` / `registrar`
